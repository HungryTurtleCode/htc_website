## Our Old Enemy &#8211; The Image Questions

<span style="font-weight: 400;">When we created the quiz controller we had the problem of the image urls displaying instead of the images themselves on image questions. We again face this problem in the results controller. Having already solved it once though, it should pose no problems to us now. We will use the Angular ng-if directive to fix the problem again.</span>

This time we want the images to display nicely but also want a border around the correct answer and if the user chose an incorrect answer, a border around the answer they selected. Both will provide feedback to the user.

<img class="aligncenter wp-image-941" src="https://res.cloudinary.com/djxscnpzf/image/upload/c_scale,w_800/v1464631777/results_images_not_displaying_viluf4.jpg" alt="images not displaying correctly" width="800" height="266" />

Like we did in the quiz controller we just duplicate the row that contains the text questions and modify it slightly to house the image questions.

<pre class="lang:xhtml decode:true" title="Image questions">&lt;div class="row"&gt;
                               
        &lt;div class="col-sm-6" ng-repeat="answer in results.dataService.quizQuestions[results.activeQuestion].possibilities"&gt;
                               	
            &lt;div class="image-answer"
                 ng-class="results.getAnswerClass($index)"&gt;
                     &lt;img ng-src="{{answer.answer}}"&gt;
            &lt;/div&gt;

    &lt;/div&gt;
&lt;/div&gt;</pre>

Now on the div with the class image-answer we will also give it an ng-class and pass in the same function we used earlier &#8211; getAnswerClass. You may say that this function returns one of two bootstrap classes, neither of which will give the images a border. You would be right.

Doing it this way cuts down on the code logic we need and we can just hook into those bootstrap classes using specificity on top of the image-answer class that the div already has to give it the required borders.

The two classes that the function can choose from is bg-success and bg-danger which are given to correct and incorrect answers respectively. We can now use those and hook into them in our own css.

Add the following to your stylesheet.

<pre class="lang:css decode:true" title="Image-answer CSS">.image-answer.bg-success{
    border: 3px solid #5ea640;
}
.image-answer.bg-danger{
    border: 3px solid #b74848;
}</pre>

## Return Of The Angular Ng-If Directive

<span style="font-weight: 400;">Again, we only want to render one block or the other &#8211; either the text answer block or the image block, but never both. This is where ng-if comes into play again. Here are the two blocks with their respective ng-if statements added onto them.</span>

<pre class="lang:xhtml decode:true " title="More of the ng-if directive">&lt;div class="row"
     ng-if="results.dataService.quizQuestions[results.activeQuestion].type === 'text'"&gt;
                               
         &lt;div class="col-sm-6" ng-repeat="answer in results.dataService.quizQuestions[results.activeQuestion].possibilities"&gt;
             &lt;h4 class="answer"
                 ng-class="results.getAnswerClass($index)"&gt;
                     {{answer.answer}}
                                      
                     &lt;p class="pull-right"
                        ng-show="$index !== results.quizMetrics.correctAnswers[results.activeQuestion] && $index === results.dataService.quizQuestions[results.activeQuestion].selected"&gt;Your Answer&lt;/p&gt;
                     &lt;p class="pull-right"
                        ng-show="$index === results.quizMetrics.correctAnswers[results.activeQuestion]"&gt;Correct Answer&lt;/p&gt;
              &lt;/h4&gt;
         &lt;/div&gt;

&lt;/div&gt;

							
&lt;div class="row"
     ng-if="results.dataService.quizQuestions[results.activeQuestion].type === 'image'"&gt;
                               
         &lt;div class="col-sm-6" ng-repeat="answer in results.dataService.quizQuestions[results.activeQuestion].possibilities"&gt;
                               		
             &lt;div class="image-answer"
                  ng-class="results.getAnswerClass($index)"&gt;
                      &lt;img ng-src="{{answer.answer}}"&gt;
             &lt;/div&gt;
         &lt;/div&gt;
&lt;/div&gt;</pre>

You can see that the statements are identical except for the type we are testing for. Image and text.

Now the app is pretty much in its completed state. We just need to give the user a way to return back to the fact page and start everything over again if they wish. Of course, we will need to reset everything when we go back as well, just so that it is a fresh start if the user wants to try again.
